qBittorrent VPS
Setup Guide

Complete guide to setting up qBittorrent-nox on a VPS for private trackers such as AvisaZ, TorrentLeech, PTP, and others. Covers installation, AWS Security Group configuration, Nginx file browser, and troubleshooting.

✓ Private Tracker Ready ✓ DHT/PeX/LSD Disabled AWS EC2 Ubuntu 22.04 / 24.04 qBittorrent-nox Nginx File Browser
01
Requirements
What you need before starting the setup.
🖥️ VPS / Server
Ubuntu 22.04 or 24.04 (Debian-based)
Minimum 1 vCPU, 1 GB RAM
Storage as needed for downloads
Root or sudo access
☁️ AWS Specific
Active EC2 instance
Access to Security Group (Inbound Rules)
Key pair (.pem) for SSH access
Elastic IP recommended
🔑 Private Tracker
Active account on the tracker (AvisaZ, etc.)
Download via .torrent file or magnet link
Passkey registered on the tracker
Ratio requirements understood
🛠️ Local Tools
SSH client (Terminal / PuTTY)
Modern web browser
FileZilla / WinSCP (optional, for SFTP)
⚠️
AWS EC2 has a Security Group acting as a second firewall layer on top of UFW. Every new port opened in UFW must also be opened in the Security Group, otherwise it remains inaccessible from outside.
02
Installation
Run the automated installer script on your VPS as root.
1
Upload and run the script
Upload setup-torrent-vps.sh to the VPS, then run it with your configuration variables.
bash
# Upload to VPS via SCP
scp -i key.pem setup-torrent-vps.sh ubuntu@SERVER_IP:~

# SSH into the VPS
ssh -i key.pem ubuntu@SERVER_IP

# Run the script with custom config
QB_PASS="StrongPassword123!" sudo bash setup-torrent-vps.sh
2
What the script does automatically
The installation process takes ~2–5 minutes depending on VPS speed.
ComponentActionDetails
qBittorrent-noxInstallFrom official PPA, latest version
User qbuserCreateDedicated user for security isolation
/opt/downloadsCreateMain download directory
systemd serviceEnableAuto-start on reboot
UFW FirewallConfigureAllow SSH, 8080, 6881
Fail2banConfigureBrute-force protection for WebUI
DHT / PeX / LSDDisabledRequired by private trackers
3
Verify the service is running
bash
# Check service status
sudo systemctl status qbittorrent-nox

# Check listening ports
ss -tlnp | grep qbit

# View logs + temporary password
sudo journalctl -u qbittorrent-nox --no-pager -n 20
💡
On qBittorrent v5.x, if the password has not been set, the service generates a temporary password visible in the logs. Look for: A temporary password is provided for this session: XXXXXXXX
03
AWS Security Group
Open the required ports in AWS — this must be done for every new port.
🚨
AWS Security Group is a firewall at the cloud level, separate from UFW. A port already opened in UFW but not in the Security Group will still result in ERR_CONNECTION_TIMED_OUT.
1
Navigate to the Security Group
AWS Console → EC2 → Instances → click your instance → Security tab → click the Security Group name → Edit inbound rules
2
Add all required ports
TypeProtocolPortSourceDescription
Custom TCPTCP80800.0.0.0/0qBittorrent WebUI
Custom TCPTCP[PEER_PORT]0.0.0.0/0Peer connection (check Options → Connection)
Custom UDPUDP[PEER_PORT]0.0.0.0/0Peer connection UDP
Custom TCPTCP80810.0.0.0/0Nginx file browser (optional)
⚠️
The peer connection port defaults to 6881, but may change if qBittorrent is set to use a random port. Always check the actual port at: Tools → Options → Connection → "Port used for incoming connections"
3
Click Save rules
Click the orange Save rules button at the bottom right. Changes take effect immediately without restarting the instance.
04
WebUI Login
Access the qBittorrent web interface.
URL
http://SERVER_IP:8080

Default credentials:
Username : admin
Password : adminadmin  # or check the temporary password in the logs
⚠️
If login with adminadmin fails, retrieve the temporary password from the logs:
sudo journalctl -u qbittorrent-nox --no-pager -n 20
Look for: "A temporary password is provided for this session: XXXXXXXX"
🔒
Change the password immediately after first login. Go to Tools → Options → Web UI → Password. The temporary password changes every time the service restarts.

06
Port Configuration
How to diagnose and fix port issues — the most common cause of stalled torrents.
bash — Port Diagnostics
# Check all ports qBittorrent is listening on
ss -tlnp | grep qbit
ss -ulnp | grep qbit

# Check UFW status
sudo ufw status numbered

# Open the peer port (replace 12021 with the actual port)
sudo ufw allow 12021/tcp
sudo ufw allow 12021/udp
sudo ufw reload

# Verify the port is open from outside:
# https://www.yougetsignal.com/tools/open-ports/
💡
The peer connection port can be random. Always check the actual port via ss -tlnp | grep qbit and make sure it is open in both UFW and the AWS Security Group. Both must be configured.
07
Nginx File Browser
Browse and download completed files directly from your browser — no SFTP needed.
1
Install and configure Nginx
bash
sudo apt install nginx -y

sudo tee /etc/nginx/sites-available/filebrowser << 'EOF'
server {
    listen 8081;
    root /opt/downloads;
    autoindex on;
    autoindex_exact_size off;
    autoindex_localtime on;
}
EOF

# Enable config and remove default
sudo ln -s /etc/nginx/sites-available/filebrowser /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t && sudo systemctl reload nginx

# Fix permissions and open firewall
sudo chmod -R 755 /opt/downloads
sudo ufw allow 8081/tcp
2
Open port 8081 in AWS Security Group
Add rule: Custom TCP → Port 8081 → Source 0.0.0.0/0 → Save rules.
3
Access the file browser
URL
http://SERVER_IP:8081
⚠️
Make sure the Default Save Path in qBittorrent points to /opt/downloads. Check at Options → Downloads → Default Save Path. If it differs, update the Nginx root directive in the config file accordingly and reload Nginx.

Troubleshooting
Common issues and solutions encountered during this setup.
❓ ERR_CONNECTION_TIMED_OUT when opening WebUI
The port is not open in the AWS Security Group. Go to EC2 → Security Groups → Edit inbound rules → add port 8080 TCP → Save rules. UFW alone is not sufficient on AWS.
❓ Invalid Username or Password on login
qBittorrent 5.x generates a temporary password per session. Check the logs:
sudo journalctl -u qbittorrent-nox --no-pager -n 20
Find the line: "A temporary password is provided for this session: XXXXXXXX". Use that password and immediately set a permanent one at Options → Web UI.
❓ Torrent Stalled — Down Speed 0 B/s even though tracker is Working
The peer connection port is not open. Steps to fix:
1. Check actual port: ss -tlnp | grep qbit
2. Open port in UFW: sudo ufw allow PORT/tcp && sudo ufw allow PORT/udp
3. Open the same port in AWS Security Group (both TCP and UDP)
4. Verify via yougetsignal.com
❓ Nginx File Browser is empty / files not showing
The qBittorrent default save path is not /opt/downloads. Check at Options → Downloads → Default Save Path. If different, update the root directive in /etc/nginx/sites-available/filebrowser to match, then run sudo systemctl reload nginx.
❓ Port shows open on yougetsignal but browser still times out
Likely a browser cache issue. Try:
1. Incognito / private mode
2. A different browser
3. Hard refresh: Ctrl+Shift+R
❓ qBittorrent does not start after a reboot
Make sure the service is enabled: sudo systemctl enable qbittorrent-nox
Check errors: sudo journalctl -u qbittorrent-nox -n 50
